\section{Executive Summary}

\subsection{The Opportunity}
The prediction market industry is projected to reach \$95.5 billion by 2035, yet current solutions face fundamental limitations. Existing platforms like Polymarket rely on centralized market creation, while decentralized alternatives like UMA suffer from governance token vulnerabilities that enable coordinated attacks with relatively small capital requirements.

\subsection{The Innovation}
Mentat Protocol solves these problems through two breakthrough technologies:
\begin{itemize}
    \item \textbf{AI Agents}: Enable permissionless market creation at scale by translating natural language prompts into technically precise prediction markets
    \item \textbf{zkTLS Integration}: Provides cryptographic proof of real-world outcomes without requiring trusted intermediaries or governance tokens
\end{itemize}

\subsection{Market Position}
Mentat uniquely combines:
\begin{itemize}
    \item Polymarket's user experience without centralized market creation
    \item Pump.fun's permissionlessness with quality controls
    \item UMA's flexibility without governance token vulnerabilities
    \item Chainlink's speed for subjective and unstructured data
\end{itemize}

\subsection{Business Model}
Revenue streams include:
\begin{itemize}
    \item Trading fees (0.5-2\% based on market quality)
    \item Market creation fees via staking requirements
    \item Premium features and analytics
    \item Data licensing and API access
\end{itemize}

\subsection{Competitive Landscape}

\subsubsection{Centralized Platforms}
\textbf{Polymarket}
\begin{itemize}
    \item Strengths: Excellent UX, high liquidity, mainstream adoption
    \item Weaknesses: Centralized market creation, regulatory risk, limited market types
    \item Market position: Dominant in crypto-native prediction markets

\textbf{Kalshi}
\begin{itemize}
    \item Strengths: US regulatory compliance, traditional finance integration
    \item Weaknesses: Limited to CFTC-approved markets, high barriers to entry
    \item Market position: Regulated US prediction markets
\end{itemize}

\subsubsection{Decentralized Platforms}
\textbf{UMA Protocol}
\begin{itemize}
    \item Strengths: Flexible oracle system, established technology
    \item Weaknesses: Governance token vulnerability (only 5M UMA needed to validate), limited user adoption
    \item Market position: DeFi-native but struggling with scalability

\textbf{Augur}
\begin{itemize}
    \item Strengths: Fully decentralized, pioneering technology
    \item Weaknesses: Poor UX, high gas costs, limited liquidity
    \item Market position: Early pioneer but largely superseded

\subsection{Multi-Agent System}

\subsubsection{Agent Types and Responsibilities}

\textbf{Creation Agent}
\begin{itemize}
    \item Parses natural language user prompts
    \item Maps requirements to zkTLS-verifiable sources
    \item Generates structured market parameters
    \item Creates unambiguous resolution criteria
\end{itemize}

\textbf{Validation Agent}
\begin{itemize}
    \item Evaluates market quality across multiple dimensions
    \item Performs duplicate detection and deduplication
    \item Assigns quality scores and risk assessments
    \item Implements configurable quality thresholds
\end{itemize}

\textbf{Resolution Agent}
\begin{itemize}
    \item Monitors target data sources continuously
    \item Generates zkTLS proofs upon trigger conditions
    \item Coordinates multi-source verification
    \item Manages dispute periods and challenges
\end{itemize}

\textbf{Dispute Agent}
\begin{itemize}
    \item Processes counter-evidence and disputes
    \item Coordinates community resolution mechanisms
    \item Handles edge cases and error correction
    \item Manages re-resolution when necessary
\end{itemize}

\subsection{Quality Scoring System}

The quality framework evaluates markets across five dimensions:

\subsubsection{Ambiguity Score (30\% weight)}
Measures resolution criteria clarity:
\begin{itemize}
    \item Precision of language and terminology
    \item Measurability of outcomes
    \item Temporal clarity and deadlines
    \item Source specification and availability
    \item Edge case coverage
\end{itemize}

\subsubsection{Relevance Score (25\% weight)}
Assesses market importance and interest potential:
\begin{itemize}
    \item Topic popularity and search volume
    \item Stakeholder impact assessment
    \item Media coverage analysis
    \item Financial significance evaluation
    \item Time sensitivity factors
\end{itemize}

\subsubsection{Intent Score (25\% weight)}
Detects malicious or spam content:
\begin{itemize}
    \item Spam pattern recognition
    \item Manipulation attempt detection
    \item Content appropriateness evaluation
    \item Creator reputation analysis
\end{itemize}

\subsubsection{Source Reliability (15\% weight)}
Evaluates zkTLS source quality:
\begin{itemize}
    \item Source tier classification
    \item Historical reliability tracking
    \item Technical compatibility verification
    \item Update frequency analysis
\end{itemize}

\subsubsection{Uniqueness Score (5\% weight)}
Prevents market duplication:
\begin{itemize}
    \item Semantic similarity detection
    \item Source overlap analysis
    \item Timeline intersection evaluation
    \item Market consolidation suggestions
\end{itemize}

\subsection{Configuration and Governance}
All quality parameters are governance-configurable, enabling:
\begin{itemize}
    \item Dynamic threshold adjustment
    \item A/B testing of scoring algorithms
    \item Community-driven parameter optimization
    \item Specialized scoring for market categories
\end{itemize}
